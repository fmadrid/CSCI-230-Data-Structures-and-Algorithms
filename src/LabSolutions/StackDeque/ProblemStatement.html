Introduction

The stack is an Abstract Data Structure (ADT) which uses in the First-In-Last-Out (FILO) principle. This particular principle is particularly useful when we want to reverse the ordering of elements in a collection in time and memory, and finding paired delimiters in a text string. The stack ADT also sees application in solving mazes when using depth-first search (DFS).

In this lab, students will implement the deque (double-ended queue) ADT using two stacks as the only instance variables. Specifically, given the Deque and Stack Java interfaces, as well as the array implementation of the Stack interface, ArrayStack. Complete the following problems.

  • Problem 1 - Class Implementation
  • Problem 2 - Driver
  • Problem 3 - Big-Oh Analysis

Write a class StackDeque that implements the Deque interface. The only two instance variables in this class should be frontStack and endStack, which are of the ArrayStack reference type. In the StackDeque class, implement each method of the Deque interface.

Hint: If you hover over the StackDeque class declaration in your implementation, you can instruct Eclipse to automatically create function stubs of each of the methods in the Deque interface.

  • int size() - Returns the number of elements in the deque.
  • boolean isEmpty() - Test whether the deque is empty.
  • E first() - Returns, but does not remove, the first element of the deque (null if empty).
  • E last() - Returns, but does not remove, the last element of the deque (null if empty).
  • void addFirst() - Inserts an element at the front of the deque.
  • void addLast() - Inserts an element at the back of the deque.
  • E removeFirst() - Removes and returns the first element of the deque (null if empty).
  • E removeLast() - Removes and returns the last element of the deque.

Hint: The frontStack instance variable represents the front end of the deque and endStack represent the back end of the deque. EAch time an element is added to the front of the deque, it is pushed onto frontStack. Each time an element is added to the end of the deque, it is pushed onto endStack. To implement removeFirst correctly, think about where the first element, considering both cases when the frontStack is and is not empty. Once you understand how to implement removeFirst, the logic for removeLast should be apparent.
